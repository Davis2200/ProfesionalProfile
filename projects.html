Banking Churn Predictive System

Machine Learning Classification | Production-Ready

XGBoost Feature Engineering Streamlit Deployment Classification

Business Context

Customer acquisition costs 5-25x more than retention. Lost customer lifetime value averages $10,000+. Traditional churn analysis is reactive—analyzing departing customers after they've left. This system is proactive: identifying at-risk accounts weeks before departure, enabling data-driven retention interventions.

Annual Impact: 850 retained customers × $10,000 LTV = $8.5M recovered value | Operational efficiency: 82% precision reduces wasted retention campaign spend

Technical Architecture

Data Pipeline: Raw customer behavior → Feature Engineering (temporal analysis, balance volatility, engagement metrics) → StandardScaler normalization → XGBoost classification → Probability scores + SHAP explainability

  • Algorithm: XGBoost (gradient boosted decision trees) for imbalanced classification with class weighting
  • Features: Account tenure, transaction frequency, balance stability, service utilization, dormancy periods
  • Scaling: StandardScaler normalization ensures algorithm convergence and fair feature comparison
  • Time-Series Handling: Rolling windows capture account health trajectories and seasonal patterns
  • Model Artifacts: JSON serialization for production inference, versioning for A/B testing

Why These Metrics Matter:

  • ROC-AUC: 0.85 — Exceeds industry baseline (0.70) by 21%; correctly ranks true churners vs stayers 85% of the time
  • F1-Score: 0.75 — Balances precision (82%) and recall (69%), avoiding both false alarms and missed cases; far superior to accuracy (92%) in imbalanced datasets
  • Precision: 82% — Only 18% unnecessary retention offers; saves $50-150 per false positive
  • Recall: 69% — Catches 69% of true churn cases; prioritizes reducing costly false negatives ($10,000+ per missed churner)
